# GENERATOR_AUDIT.md

Final audit of generated/manually maintained metadata in `expr-js` against `expr-lang/expr` v1.17.8.

## Scope

Audited items requested by handoff:

- `FUNC_TYPES`, `FUNC_TYPES_ARITY`, `TypedFuncIndex`, `IsFastFunc`
- branded type registry / branded value wrapper
- visibility metadata / visible-field-set adapter
- struct metadata / `markStruct()` / `STRUCT_TYPE`
- checker descriptors / Type descriptors
- GoLocation metadata
- time/duration metadata
- generated helper tables and manually transcribed tables

## Classification legend

- `GENERATED_FROM_GO`: machine-generated from upstream Go or a Go-equivalent generator.
- `DERIVED_FROM_GO`: manually ported from Go source with source-level traceability.
- `MANUAL`: local hand-authored data with upstream maintenance risk.
- `ADAPTER`: TypeScript/runtime shim required because Go reflection/pointers/interfaces have no direct JS equivalent.

## Manual items requiring special upgrade attention

1. `FUNC_TYPES` is the highest-risk manually transcribed table because upstream generates it and indexes are bytecode-visible.
2. Builtin descriptors/validators are hand-maintained and should be diffed against `references/expr/builtin/*.go` on every upstream bump.
3. Time layout parser/formatter is hand-ported; add Go-generated fixtures for any new layouts before changing it.

## Conclusion

The audit supports a maintainable port claim only with explicit caveats: generated Go data is not universally generated into TS. The main generator-risk item is `FUNC_TYPES`; the main adapter-risk areas are reflection replacement (`Type` descriptors/`markStruct`) and time/location modeling. All audited items have source origins and TS counterparts; none are unclassified.
