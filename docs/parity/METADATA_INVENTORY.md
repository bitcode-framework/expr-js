# METADATA_INVENTORY.md

Maintenance Parity Closure Program — Phase 1.

Inventory of metadata, tables, registries, descriptors, classifiers, and helpers that are not plain source-line ports, are manual, emulate Go reflection, or affect checker/typed dispatch/named-type/visibility/struct behavior.

## Legend

- `manual/generated`: current state in expr-js.
- `source dependency`: upstream Go source or Go behavior that this item mirrors.
- `maintenance risk`: risk when upgrading `expr-lang/expr` v1.17.8 → v1.17.9/v1.18.x.

## Highest-risk metadata

1. `FUNC_TYPES` and dependent typed-call arity.
2. `Type` / `FieldDescriptor` / `StructMeta` reflection replacement.
3. Builtin registry and per-builtin validators.
4. Time method/layout metadata.
5. `markStruct()` and visibility metadata used in strict struct parity.
6. `VALUER_METHODS` / `ValuePatcher` interface emulation.
7. Branded value wrappers for named primitive runtime equality.

## Phase 1 conclusion

The metadata surface is broader than just `FUNC_TYPES`. The true maintenance boundary is Go reflection: any upstream behavior that relies on `reflect.Type`, `reflect.StructField`, method sets, named types, or generated typed-call signatures currently crosses into manually maintained TypeScript descriptors/adapters.
